# 黃金路徑(Golden Path)使用說明

> 「黃金路徑」= 平台替開發者鋪好的、最省事又最合規的預設做法。
> 走這條路,合規是預設值,不是額外負擔。

## 開發者:三步開一個新專案

1. **生成**:在本 repo 點 **「Use this template」→ Create a new repository**。
2. **填空**:把 `README.template.md` 改名為 `README.md`,填入 `{{...}}` 佔位內容。
3. **開發**:直接寫業務邏輯。每次 PR 會自動跑機敏掃描與結構檢查(護欄)。

> 你不需要做的事:設定 CI、研究命名規範、自己防金鑰外洩、從零寫 `.gitignore`。
> 這些範本都替你預設好了。

## 範本帶來的「預設合規」一覽

| 你拿到的 | 背後對應 |
|----------|----------|
| 開箱即跑的 CI(`ci.yml`) | 護欄而非閘門 |
| 機敏資訊掃描 | ISO 27001 A.8.12 |
| 結構 / 命名檢查 | ISO 27001 A.5.37 |
| README / `.gitignore` 範本 | 降低認知負荷 |

## 治理視角:為什麼這不是「繞過變更管理」

本範本經平台 + 資安群組一次性嚴格審核後核准。開發者使用範本,等於**套用一個
已核准的「標準變更(Standard Change)」**(ISO 20000)。標準變更依框架本就
預先核准、低風險、可重複,不需每次走完整 CAB。這是變更管理最進步的形態,
不是繞過它。詳見 `COMPLIANCE_MAP.md` 第四節。

## 與護欄的關係(SoD)

- 你可以**使用**護欄(受其約束),自由開發。
- 但**修改**護欄規則(範本內的 `ci.yml`、平台的 `policies/`)需平台 + 資安
  群組審核(見 `CODEOWNERS`)。這個分離在 `[TASK-05]` 進一步以 branch
  protection 強化。
