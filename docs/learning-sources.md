# 學習來源:成為「公文↔工程」雙語翻譯者

> **原則:讀原始,不讀摘要。** 讀控制本文 / 法規本文,像讀 source code——別讀別人嚼過的。
> 進度自己標:`[ ]` 未讀 / `[~]` 讀中 / `[x]` 讀完。「我的對應」那欄寫你把它接到自己場景的筆記。

## 怎麼讀(每一條都這樣過一遍)
1. **從你公司的內控 + 稽核缺失開始**;其它標準是用來「解釋你公司為什麼這樣做、以及更好的做法長怎樣」。
2. 每讀一條控制,問:**這條對應到我的「資料欄位 + 閘門 + 物證」配方是什麼?** 讀不出來 = 還沒真懂。
3. 邊讀邊找一個內稽 / 資安的人對照「你們實際怎麼做」。**書是骨、人是肉。**
