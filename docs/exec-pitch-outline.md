# 高層簡報大綱:把治理從投影片變成系統事實

> **用途**:你(內部工程師)要對高層提案,並在「外部顧問已在場」的情境下,
> 讓決策者看出「顧問交的是地圖、你交的是會跑的路」。目標不是講贏,是**逼出一個小而具體的 yes**。
> 填空格 `[填:…]` 換成公司真實數字 / 人名 / 顧問交付物即可上場。

> **定位鐵則(全程遵守)**:**不攻擊顧問**。你是「把顧問起的頭落地、可運轉」的建設性內部人。
> 讓對比自己說話,決策者自己得出結論。你給證據,不下判斷。

---

## 0. 三個贏的支點(寫在你心裡,不是投影片)

1. **換戰場**:不在顧問主場(框架 / 成熟度曲線 / roadmap)對打。把成功定義搬到
   「能不能在**我們的真系統**上,此刻證明控制有被強制執行」。
2. **Show vs tell**:顧問帶 PowerPoint;你帶一個會 fail-closed 的真閘門 demo。這是你的核武。
3. **內部人的誠實**:你敢攤「哪些控制現在只是貼標籤」。誠實比拋光的成熟度曲線更可信,也製造急迫感。

> 你的不對稱優勢一句話:**顧問會走,你不會;顧問交報告,你交會動的系統。**

---

## 1. 投影片逐張骨架(麥肯錫式:標題即結論,一張一訊息)

> 讀法:整份**只讀標題**就是完整論證(金字塔原理)。每張結尾自問「So what?」,答不出就刪。

### S0 — Executive Summary(BLUF,30 秒看完全部)
- **標題(斷言)**:「我們有合規政策,但無法用系統證明它被執行;一個 90 天試點能把它變成可證明、且不拖慢交付。」
- 內容:三點——現況風險(不可證明)、解法(控制 by construction)、要求(90 天一個服務的試點)。
- *So what*:高層只看這張就知道你要什麼、為什麼。

### S1 — 不舒服的真相(鉤子)
- **標題**:「此刻,我們無法證明一個機密系統真的套了政策要求的更強控制。」
- 內容:一句提問——「policy 說機密資料要 X 控制;誰能現在證明線上那個系統真的有 X?」今天的答案=我們相信有人記得。
- *So what*:痛感建立。用他們的語言(風險、可證明性),不是技術。

### S2 — 為什麼會這樣(脫鉤,不甩鍋)
- **標題**:「政策活在文件、系統自己跑——中間從來沒有橋。」
- 內容:政策(Word)↔ 系統(自走)之間的斷層圖。強調「不是誰的錯,是缺機制」。
- *So what*:把問題去人格化,沒人需要防衛 → 大家能一起解。

### S3 — 代價(量化,他們的語言)
- **標題**:「這道斷層每年讓我們付出 [填:稽核救火 N 人天 / M 個發現 / 1 次事件] 的代價。」
- 內容:稽核前人工湊證據的工時、重複稽核發現、那一次「政策說 X 但系統沒做」。`[填公司數字;沒精確值給範圍]`
- *So what*:痛變成錢與風險,決策者才會動。

### S4 — 洞察(重新框架,一張一句)
- **標題**:「合規不該是事後檢查的文件,而該內建在『變更進入系統的唯一通道』上。」
- 內容:配方一行——每條控制 = ①資料欄位 ②缺它就 fail-closed 的閘門 ③自動物證。
- *So what*:這是整場的轉折點;從「管文件」翻轉成「管系統」。

### S5 — 眼見為憑(DEMO,核武,2 分鐘)
- **標題**:「看:同一條 pipeline,翻一個資料分級標籤,系統當場擋下不合規的部署。」
- 內容:現場跑 demo(腳本見第 3 節)。internal → 放行;confidential → 被擋,列出缺的高階控制。
- *So what*:政策變成「機械事實」,他們**看到**了。這一刻勝過 20 張投影片,也是顧問不會帶的東西。

### S6 — 對各角色的意義(WIIFM,讓每個人都有理由說 yes)
- **標題**:「這讓風險可證明、稽核不再救火、交付更快又更安全。」
- 內容:四欄——風險長/CISO:可證明的控制+靠例外掌舵;稽核:連續保證;CIO/交付:更快且安全;工程:不再替不是他的決策背書。
- *So what*:房間裡每個關鍵人都看到自己的好處。

### S7 — 這需要組織也微調(誠實,框成賦權)
- **標題**:「資安的角色從『擋每個變更』升級為『定政策、靠例外與指標治理』。」
- 內容:團隊拓樸(平台/產品/治理)+ management by exception。強調這是**賦權**不是威脅。
- *So what*:預先化解「這會不會搶我的權/加我的事」的抗拒。

### S8 — 我們現在在哪(誠實自評 = 燃燒平台)
- **標題**:「我們大部分控制目前是『貼了標籤、沒被系統強制』——這是可修的,我已經知道怎麼修。」
- 內容:用 `framework-conformance-assessment.md` 的「名目 vs 真」對照表(挑 5–6 條最有感的)。
- *So what*:誠實建立可信度 + 急迫感;且暗示「我有現成的修法」,不是空談。

### S9 — 決策(見第 2 節,獨立一張)

> 附錄(被問才翻):8 層參考架構、控制↔系統綁定全表、階段計劃 P0–P4、ISO 條文對照。

## 2. 決策投影片(最關鍵——多數簡報死在沒有這張)

- **標題**:「今天請你們決定:批准一個 90 天、單一服務的治理試點。」
- **給框好的選項 + 明確推薦**(不要開放式):

| 選項 | 內容 | 代價 / 風險 |
|---|---|---|
| **A(推薦)** | 90 天在 `[填:一個服務]` 做試點,3 個成功指標:`[填]` | 小、可逆、不動其他交付 |
| B | 先做一輪全面評估再說 | 又一份報告,問題依舊不可證明 |
| C | 維持現狀 | 持續稽核救火 + 風險仍不可證明(把 S3 的數字搬來) |

- **把「要的東西」做小做具體**(降低決策體積):
  - 一位高層 sponsor:`[填:誰]`
  - 一個試點團隊 + 一個服務:`[填]`
  - 一個預算上限 / 時間盒:90 天
  - 三個可量測的成功指標:`[填,如「機密部署 100% 通過分級閘門」「稽核證據產出從 N 天→自動」]`
- **製造決策時刻**:「我請各位今天決定 A/B/C,或排定 `[填:日期]` 前拍板。」

> 為什麼是小試點而非大轉型:大的他們會「再研究」(=拖死);小、可逆、有 sponsor 的,容易當場點頭,
> 而且試點成功後你就站上「land and expand」的起點。

---

## 3. DEMO 腳本(核武,務必演,≤2 分鐘)

> 用本 repo / supply-chain-demo 的真閘門(或公司代表性 pipeline)。重點是**因果看得見**。

1. 「這是我們的部署管線,跟平常一樣。」
2. 開一個部署請求,資料分級標 `internal` → 跑 → **綠燈放行**。「正常上線。」
3. 把同一個請求分級改成 `confidential`,其他完全不動 → 跑 → **紅燈擋下**,
   螢幕列出:缺 2 位 CAB 核可、缺加密物證、缺 zero-HIGH 掃描。
4. 收尾一句:「我沒改任何程式;我只翻了一個標籤。**政策從這一刻起是系統裡不可繞過的事實,不是文件裡的一句話。**」
5. (可選)秀治理儀表板:「這是管理層的駕駛艙——例外、達標率、漂移,一眼掌握,不用看每一筆。」

> 備援:現場網路 / 環境有風險 → 預錄 90 秒螢幕錄影當 fallback。**demo 出包比沒 demo 更傷**,一定要有備援。

## 4. 三大反對 + 一句話拆解(預先準備,被問就接)

| 反對 | 一句話拆解 |
|---|---|
| 「成本/資源呢?」 | 「試點只動一個服務、90 天、現有團隊;要的是決定,不是大預算。」 |
| 「會擾亂現在的交付?」 | 「只動一條串流,其他照舊;而且目標是讓合規**更無感**,不是加流程。」 |
| 「我們已經有工具/有顧問在做了?」 | 「工具和報告給了我們**地圖**;這個試點是把地圖變成**會跑的路**——剛剛的 demo 就是差別。我是來把它落地的。」 |
| 「顧問的方案不是更全面?」 | (不貶顧問)「他們的框架是好起點;我做的是最後一哩——在我們真系統上強制執行 + 留證據。兩者互補,我負責落地那段。」 |

## 5. 上台前:1:1 預先鋪路清單(IBM 式,會議室不該有驚訝)

> 鐵則:**會議室裡的每個關鍵人,都該在會前被你私下對齊過。** 公開會議只是蓋章,不是說服現場。

- [ ] 找出「有痛 + 有權」的人當 sponsor:`[填:誰]` — 先讓他點頭,由他在會議室幫你背書。
- [ ] 列政治地圖:誰有權 / 誰有痛 / 誰會擋。`[填]`
- [ ] 對「會擋的人」先 1:1:理解他的顧慮(怕搶權?怕加事?怕擔責?),會前先化解。`[填]`
- [ ] 給 sponsor 先看 demo + 決策投影片,收他的修改意見(讓他有參與感 = 變成「他的」提案)。
- [ ] 跟顧問端**保持建設性**:若場合允許,事前讓他們知道你做的是「落地他們的框架」,避免被當成踢館。
- [ ] 對每個關鍵人準備一句「他的 WIIFM」(S6 那欄的客製版)。
- [ ] 預演 demo + 備妥錄影 fallback。
- [ ] 想好「英雄是他們」的措辭:「在 `[填:sponsor]` 的支持下,我們可以…」功勞讓出去,影響力收回來。

## 6. 你 + AI 的搭配(你的不對稱產能)

- 顧問按小時計費、人力有限、交付完就走;你 + AI 能同時產出**顧問等級論述**(本 repo 的 deck / gap 分析 /
  參考架構)**和會動的真程式碼**,速度深度是按時計費追不上的。
- **分工**:你負責人與政治(sponsor、1:1、讀政治地圖、上台);我負責產出與深度
  (打磨每份產物到顧問檔次 + 做可動 demo + 補資料)。
- **餵我什麼,我給你更利的版本**:公司真實痛點數字、政治地圖、現有系統現況、顧問交了什麼 →
  我把這份大綱客製成「能直接打」的版本(填好數字、對準你公司的顧問交付物落差)。
